ПРОМТ ДЛЯ РАЗРАБОТКИ NOTEZERO (КЛОН NOTION)

Разработай веб‑приложение NoteZero — минималистичный клон Notion с тем же поведением, как на скриншотах: левый сайдбар, страницы, базы данных, чек‑листы и модальные окна создания страниц/шаблонов.

Сделай подробное ТЗ и структуру проекта, а затем предложи архитектуру (frontend + backend + база данных). Опиши ВСЕ действия пользователя в формате «при нажатии на кнопку X происходит Y». Нужно именно копировать UX Notion, но без сложных второстепенных функций.

1. Общий функционал

1.1. Workspace и навигация

У пользователя один личный workspace.

Слева вертикальный сайдбар с разделами: Search, Home, Meetings, Notion AI (можно просто AI), Inbox, блок Private со страницами (Welcome to NoteZero!, To Do List, Add new), блок Shared (Start collaborating), блок Apps (Mail, Calendar), Settings, Marketplace, Trash.

При клике по любому пункту сайдбара открывается соответствующая страница/база данных в основной части экрана.

Верх основной области: заголовок страницы, статус доступа (Private), справа кнопки: Share, звездочка (Favorite), меню «⋯».

1.2. Страницы

Каждая страница имеет:

Заголовок (редактируемый по клику).

Иконку/эмодзи слева от заголовка.

Область контента, состоящую из блоков.

Страницы могут быть обычными (текст, чек‑листы) или представлять собой базу данных (таблица задач, встречи и т.п.).

1.3. Блоки

Типы блоков минимум: текст, заголовки H1/H2/H3, чек‑лист (to‑do), буллет‑лист, нумерованный список, разделитель, цитата.

Добавление нового блока:

По Enter создаётся новый текстовый блок.

По нажатию на «+» слева от строки открывается меню выбора типа блока.

По вводу «/» открывается меню slash‑команд (тип блока + специальные элементы: Page, Database, Template и т.п.).

Блоки можно перемещать drag‑and‑drop за «⋮⋮».

По клику на «⋮⋮» показывается контекстное меню: Duplicate, Turn into (смена типа блока), Color (цвет текста/фона), Delete.

2. Экран Welcome

Сделать примерную копию страницы «Welcome to Notion!» как onboarding:

Заголовок «Welcome to NoteZero!».

Список чек‑листов (Create an account…, Download the app…, Click anywhere below and type / … и т.д., можно изменить формулировки).

Один toggle‑block («This is a toggle block…») с скрытым текстом.

Внизу справа закреплён виджет NoteZero AI (панель чата), который можно свернуть/развернуть.

3. Страница To Do List (база данных)

Сделать страницу To Do List как базу данных типа «таблица задач».

3.1. Верхняя часть

Заголовок «To Do List» и иконка (как на скрине).

Переключатель views: «To Do» и «Done» (фильтры по статусу задачи).

Справа набор иконок: сортировка, фильтр, поиск, кнопка view‑настроек и большая синяя кнопка «New».

3.2. Таблица задач

Колонки:

Checkbox (статус выполнено/не выполнено).

Name (текст задачи).

Due date (дата, Today/Tomorrow).

При нажатии на чекбокс задача помечается как выполненная и попадает во view «Done».

При нажатии на due date открывается выбор даты.

По клику по строке открывается отдельная страница задачи (модальное окно или полноэкранно) с properties и телом страницы.

3.3. Кнопка New

При нажатии на кнопку «New» в правом верхнем углу таблицы:

Создаётся новая задача со статусом «To Do» и пустым названием.

Фокус ставится в поле имени задачи.

4. Модалка создания страницы / шаблонов

При нажатии на кнопку «Add new» в блоке Private или при создании новой страницы из сайдбара:

Открывается модальное окно поверх всего (как на скрине Template Gallery).

Сверху: поле выбора области (Private / Shared) и поле поиска шаблонов.

Первые три больших блока:

Empty page.

Empty database.

Build with AI.

Ниже секция Suggested с картами шаблонов (Tasks Tracker, Projects, Document Hub и т.д., можно упростить названия, но оставить формат карточек).

Поведение:

При выборе Empty page — создаётся пустая страница с заголовком «Untitled» и пустым контентом.

При выборе Empty database — создаётся новая база данных (таблица) в текущем workspace.

При выборе любого шаблона — создаётся страница/база с предзаполненной структурой.

5. Экран Meetings (страница‑база данных)

Реализовать простую базу данных «Meetings»:

Верх: заголовок «Meetings», маленький текст «AI meeting notes».

В центре карточка с текстом + кнопка «New AI meeting note».

При нажатии на «New AI meeting note» создаётся новая запись в базе встреч и открывается её страница.

6. Экран Notion AI (NoteZero AI)

Сделать отдельную страницу с большим input в центре (как чат):

Текст‑плейсхолдер: «Ask, search, or make anything…».

Под инпутом переключатель режима (Auto / Research / All sources).

При отправке сообщения пока можно просто логировать текст или отдавать заглушку ответа (без сложной логики).

7. Share и права доступа

Для каждой страницы сверху справа есть кнопка «Share»:

При клике открывается панель справа:

Поле ввода email/username для приглашения.

Список пользователей с уровнем доступа (Full access, Can edit, Can comment, Can view).

Общий доступ: Only people invited / Anyone with the link и т.п.

Можно просто описать поведение, без сложной реализации прав на уровне БД, но заложить архитектуру под future permissions.

8. Поведение и UX (обязательно описать и реализовать)

Автосохранение всех изменений без кнопки «Save».

Возможность переименовать страницу по клику на заголовок.

В sidebar:

Перетаскивание страниц для изменения порядка.

Правый клик / «⋮» по странице: Rename, Duplicate, Move to…, Delete, Add to favorites.

Фильтрация и сортировка в базах данных:

Filter по статусу/дате.

Sort по дате или названию.

Поиск по всему workspace: при клике на Search в сайдбаре или по горячей клавише (например, Ctrl+K) появляется глобальный поиск по страницам и задачам.
