Приложение NoteZero — это веб‑workspace, где пользователь создаёт страницы и базы данных из блоков текста, списков, файлов и т.д., как в Notion. Ниже ТЗ в формате «что происходит при нажатии/действии».
​

1. Общая структура и роли
Workspace: у каждого пользователя есть личный workspace, внутри — sidebar, страницы и базы данных.
​

Страница: основная сущность, состоит из блоков, может содержать вложенные страницы и базы данных.
​
​

База данных: коллекция страниц с полями (properties), несколькими видами представлений (таблица, доска, календарь и т.д.).
​

2. Главный экран и sidebar
Когда пользователь заходит в приложение:

Отображается список последних страниц и «Home» (домашняя страница) как центр рабочего пространства.
​
​

В левом sidebar показываются:

Логотип и название workspace.

Разделы: «Home», «Все страницы», «Все базы данных», «Избранное».

Структура страниц в виде дерева (страницы и подстраницы).
​

Действия:

При нажатии на пункт в sidebar открывается выбранная страница или база данных в основной области.

При наведении на страницу в sidebar появляются иконки: «⋮» (меню), «☆» (добавить в избранное).

3. Создание страниц и управление ими
Создание новой страницы
При нажатии на кнопку «New page» в sidebar:

Создаётся пустая страница с заголовком «Untitled» и пустой областью блоков.
​

Фокус ставится в поле заголовка.

При вводе текста в заголовок он сразу сохраняется, страница появляется в sidebar.

Дополнительно:

При нажатии правой кнопкой (или «⋮») по странице:

Пункты: «Rename», «Duplicate», «Move to…», «Delete», «Add to favorites».

«Duplicate» создаёт копию страницы со всеми блоками.

«Delete» переносит страницу в корзину (логическое удаление).

Вложенные страницы
Внутри любой страницы пользователь может создать встроенную страницу блоком «Page»:

При нажатии на «+» слева от блока и выборе типа «Page» создаётся вложенная страница.
​

В sidebar она отображается как подстраница.

4. Система блоков и редактирование контента
Базовая идея: всё внутри страницы — блоки (текст, заголовки, списки, чекбоксы, цитаты и т.д.).
​

Добавление блока
При нажатии Enter в конце строки создаётся новый текстовый блок.

При наведении слева от блока показывается иконка «⋮⋮» (drag‑handle) и «+».

При нажатии на «+»:

Открывается меню типов блоков: Text, Heading 1/2/3, Bulleted list, Numbered list, To‑do, Quote, Divider, Image, File, Page и т.д..
​

При вводе «/» в начале блока открывается командное меню (slash‑commands) с фильтрацией типов блоков по тексту.
​
​

Действия над блоком
Перетаскивание: при drag‑and‑drop за «⋮⋮» блок можно перемещать вверх/вниз или вкладывать в другие блоки.

Меню блока (по клику на «⋮⋮»):

Duplicate — дублирует блок.

Turn into — смена типа (например, Text → Heading 2).
​

Color — цвет текста/фона.

Delete — удалить блок.

5. Базы данных и их страницы
Создание базы данных
Пользователь может создать базу данных:

Через кнопку «New database» или через «/table», «/board», «/calendar» и т.п. на странице.
​

При создании:

По умолчанию создаётся таблица с колонками: Name (title), Tags (multi‑select), Status (select), Created time (автоматическое поле).
​

Строки базы данных как страницы
Каждая строка таблицы — это отдельная страница базы данных, которую можно открыть в модальном окне или на весь экран.
​

При двойном клике по строке:

Открывается страница записи, где сверху показаны properties, а ниже — контент в виде обычных блоков.
​

Поля (properties)
Для базы данных доступны типы:

Text, Number, Select, Multi‑select, Date, Checkbox, URL, Email, Phone, Formula, Relation, Rollup и т.д..
​
​

При нажатии на заголовок колонки:

Можно изменить имя поля, тип, удалить поле или задать default‑значения.

6. Представления баз данных (views)
Для каждой базы данных можно создавать несколько видов: таблица, канбан‑доска, календарь, список и т.д..
​

При нажатии на выпадающий список view вверху базы:

Пункты: Table, Board, Calendar, List, Gallery.

При выборе Board:

Блоки группируются по выбранному select‑полю (например, Status) и отображаются колонками как kanban.
​

При выборе Calendar:

Элементы отображаются по полю Date в виде календаря месяца.
​

Для каждого view:

Кнопки «Filter», «Sort», «Group»:

Filter — добавить условие (например, Status = «In progress»).

Sort — сортировка по полю (например, по дате создания).

Group — группировка по полю (например, по Tags).

7. Кнопки и быстрые действия (а-ля Notion buttons)
Внутри страниц и баз данных есть специальные кнопки для автоматизации.
​
​

Примеры:

Кнопка «New task»:

При нажатии создаёт новую запись в базе данных Tasks с заранее заданными полями (Status = «To do», Date = сегодня + 1 день и т.д.).
​

Кнопка «Create template page»:

Создаёт новую страницу по заранее сохранённому шаблону (набор блоков).

Конфигурация кнопки:

При выборе кнопки «Button» в меню блоков:

Открывается настройка:

«Action: Create page in database X» или «Edit property of current page».

Предзаполненные значения полей (Status, Assignee, Due date и т.д.).
​

8. Поиск и навигация
Вверху интерфейса есть глобальная строка поиска.

При вводе текста:

Показываются результаты из страниц, баз данных и записей с подсветкой найденного текста.
​

При выборе результата:

Открывается соответствующая страница/запись.

9. Совместная работа, шаринг и права
(Это можно сделать позже, но заложить в ТЗ.)

Для каждой страницы есть кнопка «Share».

При нажатии:

Модалка с настройками:

Пригласить пользователя по email/username.

Назначить уровень доступа: View, Comment, Edit, Full access.
​

Права действуют на страницу и её подстраницы (можно переопределять на нижнем уровне).
​

10. Автосохранение и история изменений
Все действия (изменение блоков, свойств, структуры) сохраняются автоматически без кнопки «Save».
​
